# 요구명세서 (Software Requirements Specification)
## MUZIK IS MY LIFE

| 항목 | 내용 |
|---|---|
| 프로젝트 | MUZIK IS MY LIFE — 하루 한 곡 데일리 공유 플레이리스트 |
| 플랫폼 | iOS 네이티브 (Swift/SwiftUI), 게스트 입장은 App Clips |
| 백엔드 | Firebase (Firestore · Anonymous Auth · Cloud Functions · FCM) |
| 팀 | 김승완 · 김보규 · 정규호 (글로벌미디어학부) |
| 문서 버전 | v1.0 |

---

## 1. 개요

### 1.1 목적
지인 그룹이 매일 한 곡씩 유튜브 음원을 추천·공유하며 함께 데일리 플레이리스트를 만드는 iOS 앱의 요구사항을 정의한다.

### 1.2 서비스 정의
- 초대 링크 클릭만으로 **설치·가입 없이** 그룹(방)에 참여
- **1인 1일 1곡** 제한으로 선곡 품질 확보
- 매일 제시되는 **테마**에 맞춰 추천, 30자 코멘트 첨부
- 그룹 곡을 **각자 기기에서 연속 재생**
- 자정 자동 마감 후 **날짜별 영구 아카이빙**

### 1.3 용어 정의
| 용어 | 정의 |
|---|---|
| 방(Room) | 그룹 단위 공간. 고유 초대 링크를 가짐 |
| 방장(Owner) | 방을 개설한 사용자 |
| 참여자(Member) | 초대 링크로 입장한 사용자 (익명 UID + 닉네임) |
| 트랙(Track) | 등록된 유튜브 곡 1건 (곡정보 + 코멘트 + 추천자) |
| 테마(Theme) | 그날의 추천 주제 (예: "비 내릴 때 듣기 좋은 노래") |
| 마감(Close) | 자정에 당일 목록을 잠그고 아카이브로 넘기는 것 |

## 2. 기능 요구사항 (Functional Requirements)

> 우선순위: **M**(Must, MVP 필수) / **S**(Should, 이번 버전 권장) / **C**(Could, 향후 고도화)

| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-01 | 사용자는 방을 개설하고 방 이름을 입력할 수 있다 | M |
| FR-02 | 방 개설 시 고유 초대 링크(Universal Link)가 발급된다 | M |
| FR-03 | 방장은 초대 링크를 복사해 외부 메신저로 공유할 수 있다 | M |
| FR-04 | 초대 링크 클릭 시 설치·가입 없이 App Clip으로 즉시 입장한다 | M |
| FR-05 | 참여자는 닉네임을 입력해 방에 참여한다 (익명 인증) | M |
| FR-06 | 사용자는 유튜브 링크를 붙여넣거나 공유시트로 곡을 등록한다 | M |
| FR-07 | 시스템은 링크에서 videoId를 추출해 곡명·가수·썸네일을 자동 표시한다 | M |
| FR-08 | 사용자는 곡당 30자 이내 한 줄 코멘트를 작성한다 | M |
| FR-09 | 사용자당 하루 1곡만 등록 가능하다 (초과 시 차단) | M |
| FR-10 | 등록된 곡은 재생목록 최하단에 실시간으로 추가된다 | M |
| FR-11 | 사용자는 당일 목록을 등록 순서대로 연속 재생할 수 있다 | M |
| FR-12 | 재생 중 곡의 썸네일·추천자 닉네임·코멘트가 표시된다 | M |
| FR-13 | 재생 제어는 각 기기에서 독립적으로 동작한다 (동기 재생 아님) | M |
| FR-14 | 매일 방 상단에 그날의 추천 테마가 제시된다 | S |
| FR-15 | 자정까지 남은 마감 타이머가 표시된다 | S |
| FR-16 | 매일 자정에 당일 목록이 자동 마감된다 | S |
| FR-17 | 마감된 목록은 날짜별로 [과거 목록] 탭에서 다시 재생할 수 있다 | S |
| FR-18 | 사용자는 곡·코멘트를 신고할 수 있다 (심사 대응 UGC 모더레이션) | S |
| FR-19 | 새 곡 등록·마감 임박 시 푸시 알림을 전송한다 | C |
| FR-20 | 곡·코멘트에 이모지 리액션을 남길 수 있다 | C |
| FR-21 | 누적 재생목록을 외부 스트리밍 플랫폼으로 내보낸다 | C |
| FR-22 | 재생 화면 스킨(턴테이블/카세트 등)을 구매·적용한다 | C |

## 3. 비기능 요구사항 (Non-Functional Requirements)

| ID | 분류 | 요구사항 |
|---|---|---|
| NFR-01 | 성능 | 곡 등록 후 다른 멤버 화면에 3초 이내 실시간 반영 |
| NFR-02 | 성능 | 초대 링크 클릭 → App Clip 입장 화면까지 5초 이내 |
| NFR-03 | 확장성 | 방당 동시 접속 다수(수십 명) 시 목록 충돌·지연 없음 |
| NFR-04 | 보안 | YouTube Data API 키는 서버(Cloud Function)에만 보관, 클라 노출 금지 |
| NFR-05 | 보안 | Firestore Security Rules로 하루 1곡 제한·타인 데이터 수정 차단 |
| NFR-06 | 남용방지 | App Check로 비정상 클라이언트의 API 호출 차단 |
| NFR-07 | 사용성 | 가입·인증 없이 닉네임만으로 3탭 이내 곡 등록 완료 |
| NFR-08 | 호환성 | 최소 지원 iOS 16 이상 (App Clips 14+ / SwiftUI 안정성 고려) |
| NFR-09 | 심사준수 | YouTube 약관 준수 — 오디오 전용/백그라운드 추출 금지, IFrame 플레이어 노출 |
| NFR-10 | 심사준수 | UGC 신고/차단 수단 제공 (App Store 가이드라인 1.2) |
| NFR-11 | 데이터 | 마감된 목록은 날짜별로 영구 보존, 임의 삭제되지 않음 |

## 4. 제약사항 (Constraints)

- **C-01** 음원 스트리밍 자체는 유료화 불가 → 부가기능(내보내기·스킨) 중심 수익화
- **C-02** App Clip은 iOS 전용 → 안드로이드 확장 시 별도 대응 필요
- **C-03** YouTube Data API v3 일일 쿼터 한도 존재 → 메타데이터 캐싱 필요
- **C-04** 네이티브 빌드·서명·스토어 제출에 macOS + Xcode(또는 클라우드 빌드) 필요
- **C-05** App Store 심사는 선제 제출(스토어 심사 8월 1~2주차)로 리젝 대응 버퍼 확보

## 6. 인수 기준 (핵심 시나리오)

- **AC-01** 방장이 방을 만들고 링크를 공유하면, 다른 사람이 그 링크로 설치 없이 입장해 닉네임을 등록할 수 있다.
- **AC-02** 참여자가 유튜브 링크를 등록하면 곡정보가 자동 표시되고, 방의 모든 멤버 화면에 실시간으로 나타난다.
- **AC-03** 한 사용자가 같은 날 두 번째 곡을 등록하려 하면 차단된다.
- **AC-04** [재생]을 누르면 당일 곡이 순서대로 연속 재생되며 추천자·코멘트가 표시된다.
- **AC-05** 자정이 지나면 목록이 비워지고, 전날 목록은 과거 목록에서 다시 볼 수 있다.
