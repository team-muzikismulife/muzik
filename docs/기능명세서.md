# 기능명세서 (Functional Specification)
## MUZIK IS MY LIFE

> 요구명세서의 각 기능(FR)을 개발 착수 기준의 상세 동작으로 기술한다.
> 각 기능: **액터 / 선행조건 / 주요 흐름 / 예외 / 데이터**

---

## 0. 데이터 모델 (Firestore)

```
rooms/{roomId}                                           // roomId 자체가 초대코드 (링크: /r/{roomId})
  name, ownerUid, timezone("Asia/Seoul"), createdAt

rooms/{roomId}/members/{uid}
  nickname, joinedAt                                     // 하루 1곡 판정은 tracks/{uid} 존재로 (별도 필드 없음)

rooms/{roomId}/days/{yyyy-MM-dd}                          // 날짜=문서키
  themeText, closed(bool), closedAt, trackCount

rooms/{roomId}/days/{yyyy-MM-dd}/tracks/{trackId}
  videoId, title, channelTitle, thumbnailUrl, durationSec,
  submittedByUid, submittedByNickname, comment(≤30), createdAt  // 등록순=재생순

themes/{themeId}
  text
```

> **핵심 설계:** 하루가 날짜 키 문서이므로, 자정 마감은 "삭제"가 아니라 "새 날짜 문서로 전환". 아카이빙은 자동으로 이뤄진다.

## F1. 방 개설 (FR-01~03)
- **액터:** 방장 (정식 앱)
- **선행조건:** 앱 실행, 익명 인증 완료
- **주요 흐름:**
  1. 메인에서 [새로운 방 만들기] 선택
  2. 방 이름 입력 → [개설]
  3. `rooms/{roomId}` 생성(자동 ID), 개설자를 members에 등록
  4. 초대 링크(`https://<도메인>/r/{roomId}`) 생성 → [초대 링크 복사] 노출
- **예외:** 방 이름 공란 시 개설 불가 안내
- **출력:** 방 메인(F4)으로 이동, 링크 클립보드 복사 가능

## F2. 링크 입장 (FR-04~05)
- **액터:** 참여자 (App Clip 또는 정식 앱)
- **선행조건:** 유효한 초대 링크 보유
- **주요 흐름:**
  1. 초대 링크 클릭 → Universal Link 라우팅 → App Clip 즉시 실행
  2. Anonymous Auth로 익명 UID 발급
  3. 닉네임 입력 화면 → 닉네임 입력 → [입장]
  4. `rooms/{roomId}/members/{uid}`에 닉네임 저장 → 방 메인(F4)
- **예외:** 유효하지 않은/삭제된 방 링크 → 안내 후 종료. 닉네임 중복 허용(익명 UID로 구분)
- **표시:** 상단에 오늘의 테마 + 마감 타이머

## F3. 곡 등록 (FR-06~10)
- **액터:** 참여자
- **선행조건:** 방 입장, 오늘 미등록 상태(`days/{today}/tracks/{uid}` 문서 없음)
- **주요 흐름:**
  1. [곡 등록] → 유튜브/뮤직 앱 공유시트에서 선택 또는 링크 붙여넣기
  2. 클라이언트가 URL에서 `videoId` 추출 (youtu.be / watch?v= / shorts/ 대응)
  3. Cloud Function `getVideoMeta(videoId)` 호출 → 곡명·채널명·썸네일·재생시간 표시
  4. 30자 이내 코멘트 입력 → [등록]
  5. `tracks/{uid}` 생성 (문서 ID = 본인 UID → 하루 1곡 강제, 단일 쓰기)
  6. 본인 상태 '추천 완료'(당일 `tracks/{uid}` 존재로 판정), 트랙은 목록 최하단에 실시간 추가(FR-10)
- **예외:**
  - 이미 오늘 등록함 → [곡 등록] 비활성 + '추천 완료' 안내
  - 유효하지 않은 링크/비공개·삭제 영상 → 에러 안내, 등록 불가
  - 코멘트 30자 초과 → 입력 제한
  - 네트워크 오류 → 재시도 안내
- **데이터:** 트랙 1건 생성 (멤버 문서 변경 없음)

## F4. 방 메인 (FR-14~15)
- **액터:** 참여자/방장
- **구성:**
  - 상단: 오늘의 테마(FR-14), 자정까지 마감 타이머(FR-15)
  - 중앙: 당일 재생목록(등록순), [재생] 버튼
  - 하단: [곡 등록](오늘 등록 시 비활성)
  - 탭: [오늘] / [과거 목록]
- **실시간:** `days/{today}/tracks`를 createdAt 오름차순 리스너 구독 → 신규 곡 즉시 반영

## F5. 연속 재생 (FR-11~13)
- **액터:** 참여자/방장
- **선행조건:** 당일 트랙 1개 이상
- **주요 흐름:**
  1. [재생] → youtube-ios-player-helper(IFrame Player)로 첫 트랙 로드
  2. 곡 종료 이벤트 → 다음 트랙 자동 로드 (등록순)
  3. 재생 중 썸네일 + 추천자 닉네임 + 코멘트 표시(FR-12)
- **제약:** IFrame 플레이어가 화면에 노출되어야 함(NFR-09). **각 기기 독립 재생**(FR-13) — 서버 재생 동기화 없음
- **예외:** 재생 불가 영상(지역 제한 등) → 스킵 후 다음 곡

## F6. 자동 마감 & 아카이브 (FR-16~17)
- **액터:** 시스템 (Cloud Scheduler, 매일 00:00 KST)
- **주요 흐름:**
  1. Cloud Function `closeDayAndArchive` 실행
  2. 전날 `days` 문서 `closed=true`, `closedAt` 기록
  3. 오늘 `days` 문서 생성 + `themes`에서 테마 배정
  4. (FR-19 연계) 새 테마 푸시
- **조회:** [과거 목록] 탭 → `closed=true` 날짜 목록 → 선택 시 그날 목록 재생(F5 재사용)
- **데이터:** 파괴적 삭제 없음, 날짜별 영구 보존(NFR-11)

## F7. 신고 (FR-18)
- **액터:** 참여자/방장
- **주요 흐름:** 트랙/코멘트의 [신고] → 사유 선택 → 서버 기록. 임계치 초과 시 자동 숨김 또는 방장 검토
- **목적:** App Store UGC 모더레이션 요건 충족(NFR-10)

## 예외·에러 처리 원칙
| 상황 | 처리 |
|---|---|
| 네트워크 끊김 | 로컬 캐시 표시 + 재연결 시 자동 동기화 |
| 유효하지 않은 유튜브 링크 | 등록 차단 + 사유 안내 |

| 만료·삭제된 방 링크 | 입장 차단 + 안내 |
