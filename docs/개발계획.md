# 개발 계획 (Development Plan)
## MUZIK IS MY LIFE — 4주(약 1개월) 계획

> 착수일 기준 상대 일정(Day/Week). 팀 3인: 김승완 · 김보규 · 정규호
> **핵심 방침: Day 1~3 동안 전체의 50%(핵심 재생 루프)를 완성한다.**

---

## 0. "50%"의 정의

전체 MVP를 **10개 기능 블록**으로 나누고, 그중 제품의 심장인 **핵심 루프 5블록**을 3일 안에 동작시킨다.

| # | 블록 | 3일 목표? |
|---|---|---|
| 1 | 프로젝트/Firebase 세팅 + 익명 인증 | ✅ |
| 2 | 방 개설 + 초대 링크 발급 | ✅ |
| 3 | 링크 입장 + 닉네임 | ✅ |
| 4 | 유튜브 곡 등록(파싱+메타+하루1곡) | ✅ |
| 5 | 실시간 목록 + 연속 재생 | ✅ |
| 6 | 데일리 테마 + 마감 타이머 | 이후 |
| 7 | 자정 마감 + 과거 목록 아카이브 | 이후 |
| 8 | 신고(UGC 모더레이션) | 이후 |
| 9 | App Clip 패키징 + Universal Link | 이후 |
| 10 | QA·심사 준비·스토어 제출 | 이후 |

> **50% 완료 = "방 만들기 → 링크 입장 → 곡 등록 → 실시간으로 뜨고 → 연속 재생"이 실기기에서 끝까지 동작.**
> ⚠️ 3일 50%는 매우 공격적인 목표. 아래처럼 병렬 분담 + Firebase(BaaS)로 서버 구현을 최소화해야 달성 가능하다.

## 1. 🔥 Day 1~3 스프린트 (핵심 루프 50%)

### 사전 준비 (Day 0, 착수 전날)
- Apple Developer 계정, Firebase 프로젝트 생성, GitHub 협업 세팅
- Xcode 프로젝트 생성(SwiftUI), Firebase SDK(SPM) 연동, `.gitignore`

### Day 1 — 기반 + 방 생성
| 담당 | 작업 |
|---|---|
| 김승완 | Firebase 연동, Anonymous Auth, Firestore 데이터 모델/Security Rules 초안 |
| 김보규 | 방 개설 로직(`rooms` 생성) + 초대코드 발급, 데이터 액세스 계층 |
| 정규호 | 디자인 시스템(색/타이포/컴포넌트), 메인·방개설 화면 SwiftUI |
| **DoD** | 앱 실행 → 방 개설 → Firestore에 room 문서 생성 확인 |

### Day 2 — 입장 + 곡 등록
| 담당 | 작업 |
|---|---|
| 김승완 | `getVideoMeta` Cloud Function(YouTube Data API 프록시) + videoId 파서 |
| 김보규 | 곡 등록 트랜잭션(하루1곡) + 코멘트, 링크 붙여넣기 입력 |
| 정규호 | 입장(닉네임) 화면 + 곡 등록 화면 UI |
| **DoD** | 링크 없이도 방 진입 → 유튜브 링크 등록 → 곡정보 자동 표시 → Firestore 저장 |

### Day 3 — 실시간 목록 + 재생
| 담당 | 작업 |
|---|---|
| 김승완 | Firestore 실시간 리스너(목록 구독), 상태 관리 |
| 김보규 | youtube-ios-player-helper 연속 재생(곡 종료→다음 곡) |
| 정규호 | 방 메인(목록/재생/추천자·코멘트 표시) UI 통합 |
| **DoD** | ✅ **핵심 루프 end-to-end 동작** — 두 기기에서 곡 등록이 실시간 반영되고 순서대로 재생 |

> **Day 3 종료 = 50% 달성.** 데모 가능한 살아있는 제품.

## 2. 나머지 3.5주 (50% → 100%)

### Week 1 잔여 (Day 4~7) — 데일리 경험 완성
- FR-14/15 데일리 테마 + 마감 타이머
- FR-16 자정 마감 Cloud Function(Cloud Scheduler)
- FR-17 과거 목록(아카이브) 탭
- 공유시트(Share Extension)로 유튜브 앱에서 바로 등록

### Week 2 (Day 8~14) — App Clip + 심사 대응
- FR-04 App Clip 타깃 구성 + 코어 로직 공유 패키지 분리
- Universal Link / Associated Domains / `apple-app-site-association` 배포
- FR-18 신고 기능(UGC 모더레이션) — 심사 필수
- App Check 적용

### Week 3 (Day 15~21) — 고도화 + 안정화
- FR-19 푸시 알림(FCM) — 새 곡/마감 임박
- (선택) FR-20 이모지 리액션
- 예외처리 보강(네트워크·재생불가·쿼터), 메타데이터 캐싱(C-03)
- 성능: 다수 동시 접속 목록 충돌/지연 점검(NFR-03)

### Week 4 (Day 22~30) — QA + 스토어 제출
- 기능/사용성 QA, 다양한 기기·iOS 버전 테스트
- 인수 기준(AC-01~05) 검증
- App Store 심사 자료(스크린샷·설명·개인정보) 준비 → **선제 제출**
- 소프트 런칭 대비 베타 테스터(대학생 50~100명) 세팅
