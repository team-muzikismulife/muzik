# YouTube 연동 상세 설계

원칙: **videoId만이 영구 원본. 나머지 메타데이터는 전부 '캐시'로 취급한다.**
(YouTube API 정책상 API로 받은 데이터는 30일 내 갱신/삭제 의무 — §6 참고)

---

## 1. 사용하는 연동 수단 총람

| 수단 | 인증 | 쿼터 | 용도 |
|---|---|---|---|
| oEmbed (`youtube.com/oembed`) | 없음 | 없음(rate limit만) | 등록 시 빠른 미리보기, 메타데이터 갱신 |
| Data API `videos.list` | API 키 (OAuth 불필요) | 1유닛/호출, id 50개 배치 가능 | 등록 검증(임베드/지역/공개 여부), 재생시간 |
| `watch_videos?video_ids=` | 없음 | 없음 | 핸드오프 연속 재생 (비공식) |
| IFrame Player (react-native-youtube-iframe) | 없음 | 없음 | 인앱 미리듣기 |
| Data API `playlists.insert` | OAuth 민감 스코프 | 50유닛/건 | **사용 안 함** (§7) |

API 키는 **Cloud Functions에만** 둔다. 모바일 앱 바이너리에 키를 넣으면 추출·도용 가능하고 referrer 제한도 불가.

## 2. 곡 등록 파이프라인

```
[클라이언트]
 1. URL 붙여넣기/공유 → extractVideoId() (watch/youtu.be/shorts/embed/music 지원)
 2. oEmbed 호출 → 제목·채널명·썸네일 즉시 미리보기 (체감 지연 0)
 3. 코멘트 입력 → registerTrack(videoId, comment) 호출

[Cloud Functions: registerTrack]
 4. 서버 시각으로 dateKey 확정 (새벽 4시 컷오프, 클라 시계 조작 방지)
 5. videos.list(id) 1회 호출로 검증:
    - status.privacyStatus == 'public' 아니면 거부 ("비공개/삭제된 영상")
    - contentDetails.regionRestriction에 KR 차단 있으면 거부
    - snippet.liveBroadcastContent != 'none'이면 거부 (라이브는 마감 후 재생 불가 위험)
    - status.embeddable == false → 등록은 허용하되 embeddable: false 저장 (미리듣기만 스킵)
    - contentDetails.duration → durationSec 저장 (총 재생시간 표시용)
 6. tracks/{uid_dateKey} create (이미 있으면 거부 → 하루 1곡)
```

**tracks 필드 추가**: `embeddable: boolean`, `durationSec: number`, `metaRefreshedAt: timestamp`

## 3. 재생 설계

### 3-1. 핸드오프 (메인)
- 버튼: [유튜브에서 재생] → `https://www.youtube.com/watch_videos?video_ids=ID1,…` (당일 order 순, ≤50곡)
- 열리면 유튜브가 TLGG 임시 재생목록 생성 → 유튜브 앱/웹에서 연속 재생
- **유튜브가 TLGG 목록에 자체 [저장] 버튼 제공** → 사용자는 무료로 자기 계정에 재생목록 저장 가능. 우리가 만들 필요 없음. UI에 "유튜브에서 저장 버튼을 누르면 내 계정에 보관돼요" 안내 문구로 오히려 기능처럼 노출
- 스킴: https로 열기 (iOS 유니버설 링크가 유튜브 앱으로 라우팅, `youtube://` 불필요)

### 3-2. 미리듣기 (보조, 인앱)
- IFrame 플레이어 1개 + 큐 인덱스 (기기별 독립)
- `embeddable == false` 곡은 큐에서 자동 스킵 + 트랙 행에 "유튜브 전용" 배지
- `onError` (삭제/차단) → 다음 곡 자동 스킵
- 화면 꺼짐 시 정지 → "미리듣기" 포지셔닝이므로 스펙으로 수용

### 3-3. 킬스위치 (watch_videos 비공식 대비) — **구현됨 (M3)**
- Firestore `config/app` 문서: `{ handoffMode: 'watch_videos' | 'first_video' }`
- 앱 시작 시 1회 구독. 유튜브가 엔드포인트를 막으면 **배포 없이** 문서 수정만으로 전환
- `first_video` 모드: 첫 곡 watch URL만 열기 + 인앱 미리듣기를 메인으로 승격
- QA 체크리스트에 "watch_videos 동작 확인" 상시 항목 추가

**배선** — `src/lib/db.ts` `subscribeConfig` → `src/store/config.ts` `useConfigStore` → `app/_layout.tsx`에서 1회 구독.
- **안전 기본값 `watch_videos`.** `config/app` 문서가 **없어도(404)·읽기 실패해도** 기본값을 유지한다
  → 킬스위치는 평소 잠들어 있고, 관리자가 문서를 만들면 그때 활성화된다(재배포 불필요).
  데모 rules는 `config/app`에 `allow read: if true` — 공개 읽기(쓰기는 콘솔/관리자만).
- **승격 동작**(`playlist/[dateKey].tsx`): `first_video`면 [유튜브에서 재생]이 첫 곡 watch URL만 열고,
  주·보조 버튼이 뒤바뀌어 [미리듣기]가 메인(다크)으로 올라온다.

## 4. 쿼터 계산 (기본 10,000유닛/일)

| 시나리오 | 소비 |
|---|---|
| 곡 등록 검증 1건 | videos.list 1유닛 |
| 일 등록 1만 곡까지 | 쿼터 내 (SOM 5만 명 × 참여율 감안 시 수년치 여유) |
| 메타데이터 갱신 (30일 경과분) | oEmbed 사용 → 0유닛 |

쿼터 초과 시(비정상 트래픽): registerTrack이 oEmbed-only 저하 모드로 검증 없이 등록 (embeddable은 미리듣기 onError가 방어).

## 5. 메타데이터 수명주기

1. 등록 시: oEmbed(표시) + videos.list(검증) → 스냅샷 저장, `metaRefreshedAt` 기록
2. 과거 목록 열람 시: `metaRefreshedAt` 30일 경과 곡만 백그라운드에서 oEmbed 재조회 → 제목 변경 반영, 404(삭제)면 `unavailable: true` → 트랙 행에 "재생 불가" 처리 (기록 자체는 videoId로 보존)
3. 썸네일: `i.ytimg.com` 핫링크만 사용, 이미지 파일 저장 안 함 (저장 의무·용량 회피).
   **URL도 저장하지 않는다 — videoId에서 파생한다** (§5-1 참고)

## 5-1. 썸네일 파생 규칙 (2026-07-13 실측, 배포 대비 필수)

썸네일 URL은 `videoId`의 함수다. 따로 저장하면 어긋날 수 있고 실제로 어긋났다.
그런데 유튜브 썸네일 규격에 함정이 둘 있다.

| 파일 | 크기 | 비율 | 가용성 |
|---|---|---|---|
| `maxresdefault` | 1280×720 | 16:9 깨끗 | **404 잦음** (예: Ed Sheeran 'Shape of You') |
| `sddefault` | 640×480 | **4:3 레터박스** | 대체로 있음 |
| `hqdefault` | 480×360 | **4:3 레터박스** | 항상 있음 |
| `mqdefault` | 320×180 | 16:9 깨끗 | 항상 있음 |

1. **`maxresdefault`는 404가 잦다** → `sddefault` → `mqdefault` 순 폴백. (드물게 404 대신
   120×90 회색 플레이스홀더를 200으로 주므로, 로드된 폭이 320 이하면 실패로 간주한다.)
2. **`sd`/`hqdefault`는 위아래 검은 띠(레터박스)가 있다.** 그대로 카드 배경에 쓰면 띠가 보인다.
   → **항상 16:9 컨테이너에 `cover`로 그린다.** 480×360에서 실제 콘텐츠 높이는 270이고 띠가
   위아래 45px씩인데, 16:9로 cover하면 정확히 그 45px씩이 잘려나간다. 수학적으로 딱 맞는다.

구현: `src/lib/youtube.ts`(`thumbnailUrl`·`THUMB_CHAIN`) + `src/components/YoutubeArt.tsx`

## 7. 수익모델 재편

- ~~"내 유튜브 계정에 재생목록 저장" 유료 기능~~ → **폐기.** TLGG [저장] 버튼이 같은 걸 무료로 제공 (§3-1). OAuth 민감 스코프 심사 + 쿼터(건당 50유닛×곡수) 부담까지 고려하면 만들 이유 없음
- 유지: **외부 플랫폼 변환** (스포티파이/애플뮤직 내보내기, 단건 결제) — 유튜브가 못 해주는 것이라 지불 가치 있음. 곡명+채널명 기반 검색 매칭이라 Data API 의존 없음
- 유지: 플레이어 스킨/테마 판매
- 추가 검토: 마감된 플레이리스트를 이미지 카드로 만들어 인스타 스토리 공유 — 무료지만 바이럴 유입 채널
